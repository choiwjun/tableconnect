## 1. Executive Summary

### 1.1 프로젝트 개요

Table Connect는 일본 이자카야 시장을 타겟으로 한 테이블 간 익명 소통 및 선물 플랫폼입니다. QR코드 스캔만으로 같은 매장 내 다른 테이블과 연결되어 익명으로 메시지를 주고받고 실제 메뉴나 포인트를 선물할 수 있습니다.

### 1.2 핵심 가치 제안

- **사용자:** 일본 특유의 간접 소통 문화에 맞는 부담 없는 만남 방식 제공
- **가맹점:** 플랫폼을 통한 추가 주문 유도 및 부수입 창출
- **플랫폼:** 거래 수수료와 구독료를 통한 양면 시장 비즈니스 모델

### 1.3 목표

**단기 (3개월):**

- 가맹점 10개 유치
- 파일럿 검증 완료
- 핵심 기능 안정화

**중기 (6-12개월):**

- 가맹점 100개 확대
- 월간 활성 사용자 50,000명
- 손익분기점 달성

**장기 (12개월+):**

- POS 시스템 통합
- 자체 POS 개발
- 이자카야 생태계 플랫폼으로 확장

## 2. 문제 정의

### 2.1 해결하고자 하는 핵심 문제

1. **사용자 관점:** 이자카야에서 관심 있는 사람에게 말 걸기 부담스러운 문제
2. **가맹점 관점:** 손님들이 단순히 먹고 가는 것이 아니라 더 오래 머물고 추가 주문을 유도하는 문제

### 2.2 기존 솔루션의 한계

- **직접 대화:** 일본 문화상 낯선 사람에게 직접 말 거는 것이 매우 부담스러움
- **데이팅 앱:** 사전 매칭 필요, 이자카야 현장에서 즉시 사용 불가
- **SNS 교환:** 개인정보 노출, 거절 시 어색함

### 2.3 우리의 해결 방식

- **완전 익명성:** 개인정보 없이 그 자리에서만 존재하는 세션 기반
- **물리적 안전성:** 그룹으로 온 사람들이 타겟이라 1:1보다 안전
- **즉시성:** QR 스캔 후 바로 시작, 설치/가입 불필요
- **실질적 가치:** 메시지뿐 아니라 실제 선물(술, 안주) 교환 가능

## 4. 사용자 스토리

### 4.1 핵심 사용자 스토리

**[FEAT-1] 테이블 연결**

- **As a** 이자카야 방문 고객
- **I want to** QR코드를 스캔해서 내 테이블을 등록하고
- **So that** 같은 매장의 다른 활성 테이블들을 볼 수 있다

**인수 조건:**

- QR 스캔 후 3초 이내 프로필 입력 화면 표시
- 성별, 나이대, 인원수 입력 (필수)
- 닉네임 선택 입력 (없으면 랜덤 닉네임)
- 성별별 아이콘 자동 배정
- 등록 완료 후 테이블 탐색 화면으로 이동

**[FEAT-2] 익명 메시징**

- **As a** 등록된 사용자
- **I want to** 관심 있는 테이블에 익명으로 메시지를 보내고
- **So that** 부담 없이 소통을 시작할 수 있다

**인수 조건:**

- 텍스트 메시지 전송 (최대 200자)
- 이모티콘 전송
- 실시간 전달 (1초 이내)
- 상대방 차단 기능
- 부적절한 내용 AI 필터링
- 메시지 읽음 표시

**[FEAT-3] 선물하기**

- **As a** 소통 중인 사용자
- **I want to** 상대 테이블에 메뉴나 포인트를 선물하고
- **So that** 호감을 표현하고 매장 매출에도 기여할 수 있다

**인수 조건:**

- 매장 메뉴 리스트에서 선택
- 포인트 전송 (¥500-5,000)
- 결제 완료 후 즉시 상대에게 알림
- 선물 내역 화면에 표시
- 거절 불가 (받으면 자동 감사 메시지)

### 4.2 가맹점 사용자 스토리

**[ADMIN-1] 매장 관리**

- **As a** 가맹점 관리자
- **I want to** 매장 정보와 메뉴를 등록/수정하고
- **So that** 고객들이 선물할 수 있는 항목을 관리할 수 있다

**[ADMIN-2] 정산 관리**

- **As a** 가맹점 관리자
- **I want to** 일일/월별 거래 내역과 정산 금액을 확인하고
- **So that** 수익을 투명하게 관리할 수 있다

## 5. 기능 요구사항

### 5.1 MVP 범위 (Phase 1 - 3주)

### 필수 기능

1. **사용자 기능**
    - QR 스캔 → 프로필 입력 → 테이블 등록
    - 주변 테이블 탐색 (성별, 나이대, 인원수 표시)
    - 1:1 익명 메시징
    - 선물하기 (결제 연동)
    - 차단/신고 기능
    - AI 기반 부적절 언어 필터링
2. **가맹점 기능**
    - 관리자 로그인
    - 매장 정보 관리
    - 메뉴 등록/수정
    - 실시간 주문 확인
    - 일일/월별 정산 조회
3. **시스템 기능**
    - 4개 언어 지원 (일본어, 한국어, 중국어, 영어)
    - PWA 지원
    - 실시간 알림
    - 세션 관리 (자동 종료)

### 5.2 제외 사항 (Non-goals)

- ❌ 회원가입/로그인 시스템
- ❌ 프로필 사진 업로드
- ❌ 음성/영상 메시지
- ❌ 매장 간 사용자 이동 추적
- ❌ 이전 세션 기록 조회
- ❌ 그룹 채팅 (1:1만)
- ❌ POS 시스템 자동 연동 (Phase 2)

### 5.3 향후 확장 (Phase 2+)

- 매장 이벤트/쿠폰 발급 기능
- AI 기반 테이블 추천
- POS 시스템 API 연동
- VIP/프리미엄 회원 기능
- 커뮤니티 게시판

## 6. 성공 지표

### 6.1 핵심 지표 (Primary Metrics)

**노스스타 지표:**

- **가맹점 증가 속도** - 3개월 내 10개 매장 유치

**입력 지표:**

- 테이블 등록 완료율 > 80%
- 메시지 전송률 (등록자 중) > 40%
- 선물하기 전환율 (메시징 사용자 중) > 15%
- 매장당 일평균 거래 건수 > 10건
- 세션당 평균 체류 시간 > 30분

### 6.2 보조 지표 (Secondary Metrics)

- 재방문율 (같은 매장 2회 이상) > 30%
- 신고율 < 1%
- AI 필터링 차단율 < 5%
- 시스템 가동률 > 99%
- 페이지 로딩 속도 < 2초

### 6.3 비즈니스 지표

- 거래당 평균 금액: ¥2,000
- 월 구독료 납부율: 100% (자동 결제)
- 고객 획득 비용 (CAC): ¥5,000/매장
- 매장 생애 가치 (LTV): ¥300,000 (2년 기준)

## 8. 사용자 경험 (UX) 원칙

### 8.1 핵심 UX 원칙

1. **3초 원칙:** QR 스캔부터 서비스 이용까지 3초 이내
2. **제스처 중심:** 버튼 클릭보다 스와이프/탭 우선
3. **위트와 재미:** 재미있는 문구와 이모티콘으로 가벼운 분위기
4. **익명성 보장:** 개인정보 노출 최소화

### 8.2 주요 사용 시나리오

**시나리오 1: 첫 방문 (입장 직후)**

```
매장 입장 → QR 스캔 → 프로필 입력 (30초)
→ "지금 이 매장에 8개 테이블이 활성화되어 있어요!"
→ 테이블 리스트 탐색

```

**시나리오 2: 메시지 보내기**

```
관심 테이블 선택 → "안녕하세요 🍻" 전송
→ 상대방 알림 → 답장 대기
→ 대화 시작

```

**시나리오 3: 선물하기**

```
대화 중 → "한 잔 쏘기 🍺" 버튼
→ 메뉴 선택 → 결제 (30초)
→ "선물이 도착했어요!" 애니메이션
→ 상대방 감사 메시지 자동 전송

```

## 9. 리스크 및 가정

### 9.1 주요 가정

1. 이자카야 방문객의 20% 이상이 QR을 스캔할 것
2. 일본 문화상 익명 소통에 대한 거부감이 낮을 것
3. 매장은 추가 매출에 긍정적일 것
4. 20대가 주 사용층이 될 것

### 9.2 주요 리스크

| 리스크 | 확률 | 영향 | 완화 전략 |
| --- | --- | --- | --- |
| 사용자 참여율 저조 | 중 | 높음 | 첫 사용 인센티브 (¥500 포인트) |
| 성희롱/부적절 사용 | 중 | 치명적 | 강화된 AI 필터 + 신속 대응 |
| 가맹점 확보 난항 | 중 | 높음 | 3개월 무료 + 성공 사례 홍보 |
| 기술적 장애 | 낮 | 높음 | 24/7 모니터링 + 자동 복구 |
| 결제 연동 지연 | 중 | 중 | PG사 사전 협의 완료 |

### 9.3 의존성

- 일본 PG사 계약 완료 (Stripe 등)
- 파일럿 매장 10곳 섭외 완료
- 도메인 및 디자인 에셋 준비 완료
- OpenAI API 계정 (AI 필터링용)

## 10. 실험 및 학습 루프

### 10.1 검증할 가설

**가설 1:** "익명성이 보장되면 20대 사용자의 40% 이상이 메시지를 보낼 것이다"

- **관측:** 테이블 등록 대비 메시지 전송률 측정
- **학습:** 40% 미만이면 UI/UX 개선 또는 인센티브 추가

**가설 2:** "선물하기 기능이 있으면 매장 매출이 평균 20% 증가할 것이다"

- **관측:** 도입 전후 매장 일매출 비교
- **학습:** 증가폭이 낮으면 선물 추천 알고리즘 강화

**가설 3:** "일본 문화에서 간접 소통이 선호될 것이다"

- **관측:** 일본 지인 검증 완료 (매우 긍정적)
- **학습:** 실제 런칭 후 재방문율로 재검증

### 10.2 학습 계획

- **주간 리뷰:** 핵심 지표 대시보드 확인
- **월간 회고:** 가설 검증 결과 정리 및 다음 가설 수립
- **고객 인터뷰:** 월 10명 이상 사용자/매장 피드백 수집

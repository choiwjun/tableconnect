# Table Connect 프로젝트 개요서

## 1. Executive Summary

### 1.1 프로젝트 개요

Table Connect는 일본 이자카야의 테이블 간 소통을 혁신하는 차세대 플랫폼입니다. QR코드 기반의 간편한 연결 시스템을 통해 익명성을 보장하면서도 테이블 간 메시지 교환과 선물 전송을 가능하게 하여, 이자카야에서의 새로운 경험 가치를 창출합니다.

### 1.2 핵심 가치 제안

- **원활한 연결**: QR코드 스캔만으로 테이블 간 연결 완료
- **안전한 익명성**: 개인정보 공유 없이 교류 가능
- **실질적 가치**: 메뉴 아이템과 포인트의 실제 교환
- **매장 수익 증대**: 추가 주문 유도 및 재방문 고객 확보

### 1.3 목표 시장

- **1차 시장**: 도쿄·오사카·나고야의 대형 이자카야 체인 (300-500개 매장)
- **2차 시장**: 지방 도시의 인기 이자카야 (1,000-2,000개 매장)
- **타겟 고객**: 20-40대 사교적인 직장인·청년층 그룹

### 1.4 비즈니스 모델

- **거래 수수료**: 선물 거래의 15-20%
- **월 이용료**: 가맹점으로부터 ¥5,000-10,000
- **프리미엄 기능**: 광고 제거, 우선 노출 등

### 1.5 주요 성과 지표 (12개월 목표)

- 가맹 매장 수: 100개
- 월간 활성 사용자: 50,000명
- 월간 거래액: ¥10,000,000
- 거래당 평균 금액: ¥2,000

## 2. 시장 분석

### 2.1 시장 규모

- **일본 이자카야 시장**: 연간 약 ¥5.2조
- **도쿄권 이자카야 수**: 약 15,000개
- **타겟 고객층**: 20-40대 직장인 약 2,000만 명

### 2.2 시장 기회

1. **디지털화 수요**: 코로나19 이후 비대면 교류 선호 증가
2. **소셜 경험 가치**: MZ세대의 새로운 경험 추구
3. **매장 차별화**: 경쟁 심화로 차별화 요소 필요
4. **부가 수익**: 기존 POS와 별도의 새로운 수익 채널

**2.3 타겟 고객 분석**

### 최종 사용자 (End Users)

- **직장인 그룹** (35%): 회식·동료 모임, 스트레스 해소
- **청년 그룹** (40%): 친구 모임, 새로운 만남 추구
- **데이트 커플** (15%): 이색 데이트 경험
- **관광객** (10%): 현지 문화 체험

### 가맹점 (Merchants)

- **대형 체인**: 브랜드 차별화, 고객 데이터 수집
- **중소형 독립점**: 집객력 향상, 재방문율 증가
- **신규 개점**: 화제성 확보, 초기 고객 확보

**2.4 경쟁 분석**

| 구분 | 기존 POS | SNS 연동 | Table Connect |
| --- | --- | --- | --- |
| 테이블 간 소통 | ❌ | 제한적 | ✅ |
| 익명성 보장 | - | ❌ | ✅ |
| 즉시 선물 | ❌ | ❌ | ✅ |
| 매장 연동 | ✅ | ❌ | ✅ |
| 추가 주문 유도 | 제한적 | ❌ | ✅ |
| 도입 비용 | 높음 | 무료 | 낮음 |

**경쟁 우위**

- 이자카야 특화 솔루션
- 익명성과 실질적 가치 결합
- 간편한 도입 및 운영

## 3. 제품 개요

**3.1 핵심 기능**

### 1) 테이블 연결

`[QR 코드 스캔] → [테이블 등록] → [주변 테이블 탐색] → [연결 요청]`

- 매장 입구/테이블의 QR코드 스캔으로 즉시 시작
- 같은 매장 내 활성 테이블 자동 표시
- 간단한 아이콘/별명으로 익명성 보장

### 2) 익명 메시징

`[메시지 작성] → [이모티콘/스티커 선택] → [전송] → [실시간 알림]`

- 텍스트, 이모티콘, 정형 메시지 지원
- 부적절 언어 필터링 (AI 기반)
- 차단 기능으로 안전성 확보

### 3) 선물하기

`[선물 선택] → [메뉴/포인트 선택] → [결제] → [즉시 전송]`

- 매장 메뉴에서 직접 선택
- 포인트 전송 (¥500-5,000)
- 주문 자동 연동 (POS 통합)

**3.2 사용자 플로우**

### 고객 플로우

1. **입장**: 매장 QR코드 스캔 → 테이블 번호 입력
2. **탐색**: 주변 활성 테이블 확인
3. **연결**: 관심 테이블에 연결 요청
4. **소통**: 메시지 교환 또는 선물 전송
5. **퇴장**: 자동 세션 종료

### 가맹점 플로우

1. **가입**: 매장 정보 등록 → 계약
2. **설정**: QR코드 발급 → 메뉴 등록
3. **운영**: 실시간 주문 확인 → 준비 → 제공
4. **정산**: 일일/월별 정산 내역 확인

**3.3 기술적 차별점**

1. **초경량 아키텍처**
    - PostgreSQL + node-cache 조합
    - 초기 인프라 비용 최소화
    - 확장 가능한 구조 설계
2. **실시간 처리**
    - WebSocket 기반 즉시 알림
    - 매장 내 지연시간 < 1초
3. **보안**
    - 세션 기반 익명성
    - 개인정보 최소 수집
    - PCI-DSS 준수 결제
4. **확장성**
    - Phase별 단계적 확장 계획
    - 클라우드 네이티브 설계

- **데이터 기반**: 가맹점에 유용한 인사이트 제공
